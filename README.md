# [모바일 응용 소프트웨어 기말 프로젝트 제출]

* **학번:** 242147
* **이름:** 김지원
* **분반:** 1분반

## 1. 앱 기획 및 테마 설명
* **앱 이름:** (예: Smart Book Market)
* **테마:** 스마트폰 속 나만의 작은 도서 쇼핑몰
* **기획 의도:** 필수 과제 요구사항을 충족함과 동시에, 실무 쇼핑몰 앱과 유사한 사용자 경험(UX)을 제공하는 다목적 도서 마켓 목업(Mock-up) 앱입니다.
   DrawerLayout과 BottomNavigationView를 활용하여 화면 공간을 효율적으로 사용하고, SharedPreferences와 Intent를 통해 각 화면 간의 데이터가 자연스럽게 이어지도록 기획했습니다.

## 2. 개발 및 실행 환경
* **IDE:** Android Studio 2025.3.1 Patch 1
* **언어:** Kotlin
* **Min SDK:** 24
* **Target SDK:** 36
* **참고:** 프로젝트 내의 build.gradle 파일이 포함되어 있으므로 Android Studio에서 프로젝트를 열고 Sync를 진행하시면 필요한 라이브러리가 자동 구성됩니다.

## 3. 주요 액티비티(Activity) 및 XML 구성
본 앱은 기능별로 클래스와 레이아웃이 철저히 분리되어 있습니다.

* **[인트로 및 메인 허브]**
  * **Splash & MainActivity:** 앱 최초 실행 시 나타나는 스플래시 화면을 거쳐 진입하는 메인 허브입니다. ViewPager2를 이용한 상단 배너 슬라이드, GridLayout을 이용한 메뉴 버튼, 그리고 DrawerLayout을 활용한 측면 네비게이션 메뉴를 제공합니다.
  * 하단 메뉴바(BottomNavigationView) 중앙에는 장바구니 속 책의 개수를 띄워주는 Badge 기능이 포함되어 있습니다. 

* **[도서 목록 및 필터링 기능]**
  * **BookListActivity:** 판매 중인 책 목록을 제공합니다. RecyclerView와 BookAdapter, Book(데이터 클래스)을 이용해 목록을 구현했습니다. 
  * 메인 배너에서 '이달의 추천도서'를 클릭하거나, 하단바에서 '장바구니'를 클릭했을 때 Intent Extra(꼬리표)를 받아 조건에 맞는 도서만 필터링하여 재활용하는 로직이 적용되어 있습니다.

* **[도서 상세 및 부가 기능]**
  * **BookDetailActivity:** 선택한 도서의 상세 정보(표지, 제목, 저자, 가격 등)를 확인합니다.
  * **기능 1 (AlertDialog):** 장바구니 버튼 클릭 시 구매 확인용 커스텀 알림창 팝업이 발생합니다.
  * **기능 2 (SharedPreferences):** 해당 화면에 진입하는 순간의 도서 정보를 로컬에 저장하여, 메인 화면의 '나의 페이지'에서 최근 본 도서로 바로 이동할 수 있도록 연동했습니다.

* **[로그인 목업 화면]**
  * **LoginActivity:** State Selector(상태 선택자)를 활용하여 입력창 터치 시 테두리 색상이 변경되는 디테일한 UI 반응형 목업 로그인 화면입니다.

## 4. 앱 실행 및 조작 방법
1. 압축을 푼 폴더를 Android Studio에서 `[File] -> [Open]`으로 엽니다.
2. 하단 상태 표시줄에서 Gradle Sync가 완료될 때까지 대기합니다.
3. 안드로이드 에뮬레이터를 실행하여 앱 시작 화면으로 진입합니다.
4. 메인 화면의 스와이프 배너, 격자 메뉴 버튼, 상단 툴바 메뉴(더보기), 하단 네비게이션 바 등 다양한 UI를 터치하여 앱을 탐색할 수 있습니다.
5. 미구현된 일부 부가 기능 버튼들은 안전한 예외 처리를 위해 Toast 메시지가 출력되도록 설계했습니다.

## 5. 참고 사항 및 주의사항 (Troubleshooting)
* **하드코딩 방지 처리:** 코드 채점 기준을 완벽하게 충족하기 위해, 소스 코드(.kt) 및 레이아웃(.xml) 내의 모든 텍스트 데이터는 `res/values/strings.xml` 리소스로 철저히 분리하여 `getString()`으로 호출하도록 작업했습니다.
* **UI 최적화:** 액티비티 이동 시 데이터가 꼬이거나 앱이 종료되는 것을 방지하기 위해 Context 참조 타이밍(`onCreate`, `onResume` 등)과 Intent 생명 주기를 엄격하게 관리했습니다.
